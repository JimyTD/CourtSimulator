## 一、产品定位

用户扮演皇帝，提出一个朝政议题（如"西北用兵""减免赋税""修建运河"），朝中官员各持立场轮番奏对，用户最终拍板裁决。

**核心乐趣**：
- 每个官员立场鲜明、说话方式有辨识度
- 官员之间会互相呛声、引经据典，且考虑彼此品级
- 皇帝可以自由发挥，百官不考虑皇帝立场

## 二、角色系统

### 2.1 默认官员（6 + 1）

| 角色 | 立场 | 说话风格 |
|------|------|---------|
| 户部尚书 | 财政保守派 | 开口必谈银两，反对一切花钱 |
| 兵部尚书 | 强硬鹰派 | 凡事先想打，好战不服软 |
| 礼部尚书 | 祖制保守派 | 引经据典，反对任何变革 |
| 工部尚书 | 技术实用派 | 只聊能不能造出来，不谈政治 |
| 御史大夫 | 监察找茬派 | 专门挑毛病，谁的方案都能喷 |
| 清流翰林 | 道德理想派 | 引圣人之言，追求仁义正道 |
| 丞相（内阁首辅） | 中立归纳者 | 总结各方意见，推动皇帝拍板 |

### 2.2 用户封官（自定义角色）

用户可以创建自定义官员，字段如下：

| 字段 | 说明 | 限制 |
|------|------|------|
| 官名 | 系统预设列表 或 完全自创（如"总统"） | 任意文本 |
| 品级 | 从 9 品到 1 品 | 必选，影响游戏逻辑 |
| 性格特点 | 用户自由填写 | 自由文本 |
| 说话风格 | 用户自由填写 | 自由文本 |

**AI 润色流程**：用户填写完后，后端调用 LLM 将用户文本润色成标准 prompt 格式（对标默认官员水准），用户可预览并确认。

### 2.3 品级影响逻辑

品级（1 品最高，9 品最低）影响官员在**第二轮反驳**时的语气倾向：

| 场景 | 行为 |
|------|------|
| 反驳比自己高 3 级以上的官员 | 措辞委婉，先铺垫再否定，"下官斗胆" |
| 反驳同级官员 | 直接对峙，不客气 |
| 反驳比自己低的官员 | 居高临下，可以不留情面 |
| 品级 7-9 品 面对 1-2 品 | 可能完全沉默，显示"臣无奏" |

品级信息通过 prompt 注入：
```
你是从五品御史。当前朝堂中有一品丞相、二品尚书。
与品级高于你的官员交流时，措辞需适当恭敬...
```

## 三、辩论流程

### 3.1 核心流程

```
皇帝输入议题
     │
     ▼
后端并行调用所有参与官员（第一轮）
     │
     ▼
各官员完整回复通过 WebSocket 推送前端
（后端流式接收攒完整段，再推给前端）
     │
     ▼
前端打字机动画展示各官员发言
     │
     ▼
所有官员发言完毕 → 触发第二轮
     │
     ▼
后端并行调用所有官员（第二轮，context 包含所有人第一轮发言）
     │
     ▼
第二轮发言展示完毕 → 丞相总结
     │
     ▼
皇帝拍板（选方案 / 自由输入裁决）
```

### 3.2 轮次设计

- **MVP 默认 2 轮**（第一轮陈述 + 第二轮反驳）
- 设置面板可调 1-3 轮
- 第 N 轮的 context = 议题 + 前 N-1 轮所有人的发言

### 3.3 沉默处理

品级差距大时，低品级官员第二轮可能沉默：
- 前端显示：该官员卡片显示「臣无奏」
- 沉默概率由 prompt 控制（不是硬编码概率，由模型自行决定）

### 3.4 后续扩展（P1）

- **皇帝追问**：点名某官员，该官员针对皇帝问题单独奏对
- **皇帝打断**：随时终止辩论，直接裁决（需要 WebSocket 双向通信）

## 四、显示设计

### 4.1 打字机效果

- 后端：流式接收 LLM 输出，攒成**完整段落**后一次性推给前端
- 前端：收到完整文本后，启动打字机动画逐字展示
- 速度：设置面板可调（快/慢）

**不做半句话流式显示**，必须完整文本到达再开始动画。

### 4.2 朝堂布局（建议）

```
┌─────────────────────────────────────────┐
│              当前议题                    │
│         「西北用兵，如何应对？」          │
├─────────┬─────────┬─────────┬───────────┤
│ 户部尚书 │ 兵部尚书 │ 礼部尚书 │ 工部尚书  │
│  正在思考│  臣以为..│  臣无奏  │  ...     │
├─────────┴─────────┴─────────┴───────────┤
│              御史大夫 · 清流翰林          │
├──────────────────────────────────────────┤
│              丞相总结（第二轮后出现）      │
├──────────────────────────────────────────┤
│  皇帝：[输入裁决...]         [御批]       │
└──────────────────────────────────────────┘
```

## 五、设置面板

| 配置项 | 选项 | 说明 |
|--------|------|------|
| 发言长度 | 短（~100字）/ 中（~200字）/ 长（~350字） | 注入 prompt |
| 辩论轮次 | 1 / 2 / 3 轮 | 默认 2 轮 |
| 文言文程度 | 偏现代 / 偏文言 | 默认偏现代 |
| 参与官员 | 多选勾选 | 默认全选 |
| 打字机速度 | 快 / 慢 | 前端动画速度 |
| AI 配置 | 见下 | 高级用户 |

### AI 配置子面板

| 字段 | 说明 |
|------|------|
| 模型提供商 | Gemini / DeepSeek / GLM4 / OpenAI / 自定义 |
| API Key | 用户输入，存 localStorage，不上传服务器 |
| Base URL | 高级选项，支持代理或私有部署 |
| 模型名称 | 下拉预设 + 手动输入 |

## 六、AI 接口设计

### 6.1 默认调用链（Fallback）

```
用户有自己的 Key → 优先使用用户 Key
     ↓ 没有
DeepSeek V3（服务端默认 Key，免费额度）
     ↓ 429 / 余额不足
GLM4-Flash（服务端备用 Key，几乎无限免费）
```

### 6.2 服务端环境变量

```
DEEPSEEK_API_KEY=xxx     # 主力
GLM4_API_KEY=xxx         # 兜底
```

### 6.3 费用策略

- 服务端默认链路：**零成本**（均为免费 API）
- 用户自带 Key：用户自己承担，与服务方无关
- 限流：架构预留接口，暂不实现
