# Changelog 面向用户编写规则

## 📌 核心原则

**Changelog 面向用户，不面向开发者。只写用户能感知的变化，删除所有技术细节。**

---

## ✍️ 内容要求

### ✅ 应该写的内容

- **功能新增** — 用户能直接体验的新功能
- **Bug 修复** — 用户遇到的问题已解决（用用户语言描述）
- **界面调整** — 外观、交互的改进
- **性能提升** — 用户能感知的速度提升
- **用户体验改善** — 更方便、更直观等

### ❌ 禁止写的内容

- **技术实现细节** — 不写文件名、函数名、类名
- **代码变动** — 不写"重构 auth.js""优化 reducer"等
- **配置项修改** — 不写 docker-compose.yml、nginx.conf 等
- **依赖更新** — 不写"升级 React 18.3""降级 Vite 5"等
- **构建工具细节** — 不写 Webpack、Vite 等工具相关的改动
- **后端实现** — 不写数据库、API 层等内部实现

---

## 🎯 对比示例

### ❌ 错误（技术细节）

```markdown
## 2026-03-20
- 修复术语管理器初始化时序问题（`T.placeholder` 未就绪导致 JS 报错）
- `terminology.js` 默认配置补全 `placeholder`、`message` 等节点
- 修复服务器 MongoDB 认证失败（`docker-compose.yml` 被覆盖）
- 升级 Vite 从 6.0 到 5.4.21，解决 React CJS 兼容性
- 重构 TerminologyContext，使用 window 全局单例模式
- 修复 Nginx 缓存策略，将 APK 缓存时间改为 10 分钟
```

### ✅ 正确（用户视角）

```markdown
## 2026-03-20
- 修复页面偶尔加载报错的问题
- 修复某些用户无法登录的问题
- 优化了应用加载速度
- 改进了对话框的显示效果
```

## 📝 格式规范

### 标准格式

```markdown
# 更新日志

## YYYY-MM-DD
- 简短描述1
- 简短描述2
- 简短描述3

## YYYY-MM-DD
- 简短描述4
- 简短描述5
```

### 具体要求

1. **新条目追加在最顶部**（`# 更新日志` 标题之后）
2. **按日期从新到旧排列**
3. **日期格式统一** 为 `YYYY-MM-DD`
4. **每条一行**，不用技术术语
5. **语言简洁** — 用户能一目了然

## 💡 写作指南

### 步骤 1：列出技术改动

```
- 修复 auth.js 中的 token 过期逻辑
- 优化 MongoDB 查询性能
- 更新 Express 中间件配置
- 修复 React Component 的生命周期问题
```

### 步骤 2：转换为用户语言

问自己："普通用户会怎么表述这个变化？"

```
- 修复用户有时无法登录的问题
- 改进了应用响应速度
- 修复某些操作延迟显示的问题
- 优化界面在不同设备上的显示效果
```

### 步骤 3：去掉非必要信息

- ✅ 保留用户能感知的变化
- ❌ 删除内部实现、工具、配置等

---

## 📋 写作检查清单

在提交 Changelog 前，必须确认：

- [ ] 每条描述都是用户能理解的吗？
- [ ] 是否有文件名、函数名等技术词汇？
- [ ] 是否有 Docker、Nginx、MongoDB 等工具名称？
- [ ] 是否有"代码""重构""优化算法"等开发术语？
- [ ] 是否过于冗长？能简化吗？
- [ ] 是否需要补充说明某个用户可能感兴趣的变化？

## 🔄 维护建议

### 何时更新 Changelog

- 发版前（必须）
- 重要 bugfix 后（推荐）
- 功能上线后（推荐）

### 何时不需要更新

- 内部重构或代码优化（用户感受不到）
- 依赖版本升级（用户无感知）
- CI/CD 配置修改（后端工具更新）

### 定期审查

- **每周**：检查是否有遗漏的用户可感知的变化
- **发版前**：全面审阅，确认所有条目都符合用户视角
- **事后**：如发现技术细节漏网，下次改进

## ⚠️ 常见误区

### 误区 1：把 Changelog 当做 Git commit log

**错误**：
```
- Refactor authentication module
- Update MongoDB connection pooling
- Fix React hook dependency array
```

**正确**：
```
- 改进登录安全性
- 修复某些用户连接超时的问题
- 优化应用启动速度
```

### 误区 2：Changelog 太长，没人想看

**错误**：
```
- 修复 auth.js 中的 JWT token 过期检查逻辑（之前未考虑 refresh token 的有效期）
- 优化 MongoDB 查询（从 O(n²) 改为 O(n log n)，使用了新的索引策略）
```

**正确**：
```
- 修复某些长时间未操作的用户被强制登出的问题
- 改进了应用加载大列表时的速度
```

### 误区 3：技术词汇出现

**错误**：
```
- 修复 Redux reducer 中的状态管理问题
- 更新 Docker Compose 配置
- 升级 Webpack 到 5.x
```

**正确**：
```
- 修复某些操作后页面数据不更新的问题
- 改进服务器部署流程
- 提升应用构建性能
```

## ✅ 审阅建议

### 提交前的自审清单

1. **读一遍** Changelog，假装自己是用户
2. **问自己** — "我能理解这个变化吗？"
3. **检查** — 有没有技术词汇？
4. **简化** — 能更简洁吗？
5. **对齐** — 是否符合项目风格？

### 提交后不修改历史

```
❌ 已提交的 Changelog 条目不要修改或删除
❌ 如需更正，在新条目中说明（而不是改历史）

✅ 正确：在新条目补充说明
## 2026-03-21
- 修正：上版本"性能优化"主要针对列表加载，感谢反馈
```
