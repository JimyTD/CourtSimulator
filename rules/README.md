## 🎯 规则分类

### 🔴 核心协作规范（必读）
- **问题排查效率原则**（01）— 在任何 AI 辅助项目中都需要
- **技术选型预警**（02）— 避免反复打补丁的陷阱

### 🟡 工程规范
- **Git 提交授权**（03）— 保护代码安全
- **文案配置驱动**（04）— 提升可维护性和国际化能力
- **文档分层管理**（05）— 保持项目整洁
- **用户面向 Changelog**（06）— 改善用户体验和沟通

### 🟢 环境优化
- **Windows 效率优化**（07）— 专为 Windows 开发环境

## 🔧 使用方式

### 在新项目中应用规则

1. **选择相关规则** — 根据项目技术栈选择
2. **复制到新项目** — 复制到 `.codebuddy/rules/` 目录
3. **改为 .mdc 扩展** — 改为 `.mdc` 扩展名（CodeBuddy 规则格式）
4. **根据项目调整** — 修改路径、工具、配置等项目特定的细节

### 规则优先级说明

- **🔴 最高/高** — 强制执行，几乎所有项目都需要
- **🟡 中高/中** — 根据项目类型选择应用
- **🟢 低** — 可选，只在特定场景适用
