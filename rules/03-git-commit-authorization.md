# Git 提交授权机制

## 🎯 核心原则

**AI 助手不可以主动提交代码，但在用户明确指令下可以执行 Git 操作。**

---

## 📋 操作规则

### ❌ 禁止主动提交

在以下情况下**严格禁止** AI 助手自动执行 Git 操作：

- ❌ 完成代码修改后自动提交
- ❌ 用户没有明确要求提交
- ❌ 用户只是说"完成了"、"好了"等模糊表述
- ❌ AI 认为"应该提交了"而主动提交

### ✅ 允许经授权的提交

在以下情况下**允许** AI 助手执行 Git 操作：

- ✅ 用户明确说"提交到 Git"
- ✅ 用户明确说"推送到远程"
- ✅ 用户明确说"commit 并 push"
- ✅ 用户明确说"帮我提交"
- ✅ 用户使用其他**明确的 Git 操作指令**

---

## 🎓 明确指令的判断标准

### 明确指令包含但不限于

```
- "提交到 Git"
- "推送到 Git 远端"
- "commit 这些改动"
- "push 到 main 分支"
- "帮我 git commit"
- "提交并推送"
- "merge 到主分支"
```

### 模糊表述（不算明确指令）

```
- "好了"
- "完成了"
- "可以了"
- "没问题"
- "继续"
- "谢谢"
```

---

## ✅ 只读操作（始终允许）

AI 助手**始终可以**执行以下只读操作，无需明确授权：

```bash
# 查看状态
git status
git log
git diff

# 查看信息
git branch
git remote -v
git show
git blame

# 查看历史
git log --oneline
git reflog
```

---

## 📖 正确的工作流程

### 场景 1：用户未明确要求提交

#### AI 助手完成修改后

1. ✅ 列出修改的文件
2. ✅ 告知用户已完成
3. ✅ 可选运行 `git status` 显示改动
4. ✅ **提示用户可以提交**（提供命令模板）
5. ❌ **不主动执行 git commit/push**

**示例对话**：
```
我已完成修改：
- routes/auth.js
- public/login.html

如需提交，请运行：
```bash
git add .
git commit -m "描述"
git push origin main
```

或者直接告诉我"提交到 Git"，我来帮你执行。
```

### 场景 2：用户明确要求提交

#### 用户说："提交到 Git" / "帮我推送" / "commit 并 push"

1. ✅ AI 识别为明确指令
2. ✅ 执行 `git add`、`git commit`、`git push`
3. ✅ 使用合适的提交信息
4. ✅ 告知用户提交结果

**示例执行**：
```bash
git add .
git commit -m "添加登录功能"
git push origin main
```

**结果反馈**：
```
✅ 登录功能已提交并推送到 Git！
   提交信息：添加登录功能
   推送到：origin/main
```

### 场景 3：用户完成后模糊表述，再明确指令

```
用户：添加登录功能
AI：[完成代码]
AI：✅ 已完成！[提示可以提交]

用户：好了
AI：[不执行提交]
AI：修改已完成。需要提交到 Git 吗？

用户：提交到 Git
AI：[执行 git add, git commit, git push]
AI：✅ 已提交并推送！
```

---

## 💡 设计理由

这个规则平衡了：

- ✅ **用户的自主控制权** — 用户决定何时提交
- ✅ **AI 协助的便利性** — 用户明确要求时可快速提交
- ✅ **代码安全性** — 避免意外提交或提交不完整代码
- ✅ **工作流灵活性** — 支持不同的开发工作流

同时教会用户：
- 如何与 AI 协作
- 何时需要明确指令
- Git 操作的重要性

---

## ⚠️ 特殊情况

### 用户明确授权的自动提交

在某些长期项目中，用户可能会事先授权："以后每次修改完直接提交"。

**处理方式**：
1. 获取用户的**书面授权**（在 MEMORY.md 或项目文档中记录）
2. 每次提交前仍然**列出修改文件清单**
3. 提交时**说明理由**
4. 用户可随时撤销授权

---

## ✅ 检查清单

完成代码修改后，必须确认：

- [ ] 用户的指令是否明确包含 Git 操作词？
- [ ] 是否只是说"好了""完成了"等模糊词？
- [ ] 是否存在已授权的自动提交规则？
- [ ] 如需提交，是否列出了修改文件？
- [ ] 是否提供了清晰的提交信息模板？
